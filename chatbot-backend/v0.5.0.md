# Version 0.5.0 Implementation Plan

## 1. Global AI Memory System
- **Database Update:** Create a new `memories` table linked to `user_id`.
- **Backend Flow:** Create a background task that periodically summarizes chat contents using the LLM to extract key facts about the user and saves them.
- **Frontend UI:** Add a "Memory" section in the dashboard where users can view, edit, and delete their saved memories (similar to ChatGPT).
- **Context Injection:** Modify the chat endpoint to retrieve these memories and prepend them to the system prompt so the AI always "remembers."

## 2. Auto-generate Chat Titles
- **Backend Flow:** Upon receiving the first message in a new conversation, dispatch an asynchronous LLM request to summarize the prompt into a short title.
- **Database Update:** Save the generated title to the conversation record.
- **Frontend UI:** Display this dynamic title in the sidebar instead of a generic string.

## 3. Reread Current Chat History
- **Backend Flow:** Update `routes/chat.py` to retrieve the last N messages associated with the `conversation_id`.
- **Inference:** Format these previous messages and pass them into the Ollama payload so the AI has continuous context of the conversation.

## 4. Web Search Integration
- **Frontend UI:** Add a "Search Web" toggle near the chat input box.
- **Backend Flow:** If toggled on, intercept the prompt, perform a web search via an API (e.g., DuckDuckGo or Tavily), extract top snippets, and feed them into the LLM context.

## 5. Sidebar Menu ("...")
- **Frontend UI:** Add a `...` button next to the `x` (delete) button on each chat item.
- **Features:** Open a dropdown menu allowing users to view chat details or manually rename the conversation.

## 6. Dynamic Timeouts & Error Messages
- **Backend Flow:** Refactor `generate_response` in `ollama_service.py` to accept a dynamic timeout parameter based on the model ID.
- **Timeout Mapping:**
  - `phi`: 180 seconds (3 min)
  - `7b` models: 300 seconds (5 min)
  - `14b` models: 450 seconds (7.5 min)
  - `32b` models: 750 seconds (12.5 min)
- **Error Handling:** Update the 504 Gateway Timeout exception text in `routes/chat.py` to return `"Error TIMEOUT"`.

---

## ❌ Not Possible / Out of Scope
**On-the-fly Fine-Tuning / Training via Toggle:**
Fine-tuning large language models requires a dedicated, resource-intensive machine learning pipeline (like PyTorch or MLX) and creates new model weights. It cannot be done dynamically in real-time per chat on an M2 Mac Air via Ollama. 
*Workaround:* The Global AI Memory System (Feature 1) will successfully simulate this by ensuring the model remembers user facts via prompt injection.
